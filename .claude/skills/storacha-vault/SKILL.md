# Skill: storacha-vault
Encrypt sensitive agent data and upload it to Storacha with Lit Protocol access gating.

## Trigger Phrases
- "Securely persist agent memory"
- "Archive deliberation log"
- "Encrypt and upload to Storacha"
- "Store with Lit access control"

## Steps

1. **Initialize the encrypt-upload client:**
   Use `createGenericLitAdapter` from `@storacha/encrypt-upload-client/factories.node`
   with a Lit client (`nagaDev` network) and auth manager.

2. **Define Lit Access Control Conditions:**
   ACCs are auto-generated by the encrypt-upload-client based on the Storacha space.
   For custom NEAR-gated ACCs, override via `EncryptionConfig`.

3. **Encrypt and upload:**
   Call `encryptedClient.encryptAndUploadFile(blob, encryptionConfig)` — returns a CID.

4. **Retrieve and decrypt:**
   Call `encryptedClient.retrieveAndDecryptFile(cid, decryptionConfig)` — returns a DecryptionResult.

5. **Log CID on-chain (optional):**
   Call `near.functionCall({ methodName: 'log_cid', args: { cid, context_id } })`.

## Implementation
- `coordinator-agent/src/storacha/vault.ts` — Core encrypt/upload/decrypt functions
- `coordinator-agent/scripts/test-vault.mjs` — E2E test script

## Environment Variables Required
- `STORACHA_AGENT_PRIVATE_KEY` — Ed25519 private key (from `storacha key create`)
- `STORACHA_DELEGATION_PROOF` — Base64-encoded UCAN delegation CAR
- `LIT_NETWORK` — Lit network (default: `nagaDev` for testing)
